Guía de Comandos Git para Subir un Repositorio

Comandos Básicos

1. Inicializar el repositorio

git init

Inicializa un nuevo repositorio Git en el directorio actual.

2. Agregar archivos al área de preparación

git add .

Agrega todos los archivos modificados al área de preparación.

3. Realizar un commit

git commit -m "Primer commit"

Crea un commit con los cambios agregados al área de preparación.

4. Vincular el repositorio remoto

git remote add origin URL_DEL_REPOSITORIO

Vincula el repositorio local con el repositorio remoto en GitHub.

5. Subir los cambios

git push -u origin main

Sube los commits desde el repositorio local al repositorio remoto.

Errores Comunes y Soluciones

Error: Rejected (fetch first)

To https://github.com/pmacchi/projectOne
 ! [rejected]        main -> main (fetch first)
error: failed to push some refs to 'https://github.com/pmacchi/projectOne'
hint: Updates were rejected because the remote contains work that you do not
hint: have locally. This is usually caused by another repository pushing to
hint: the same ref. If you want to integrate the remote changes, use
hint: 'git pull' before pushing again.
hint: See the 'Note about fast-forwards' in 'git push --help' for details.

Solución:

git pull origin main

Trae los cambios del repositorio remoto y fusiónalos con tu rama local.

Error: Non-fast-forward

To https://github.com/pmacchi/projectOne
 ! [rejected]        main -> main (non-fast-forward)
error: failed to push some refs to 'https://github.com/pmacchi/projectOne'
hint: Updates were rejected because the tip of your current branch is behind
hint: its remote counterpart. If you want to integrate the remote changes,
hint: use 'git pull' before pushing again.
hint: See the 'Note about fast-forwards' in 'git push --help' for details.

Solución:

git pull --rebase origin main

Trae los cambios del repositorio remoto y rebasea tu rama local.
